# Supabase CLI Migration Workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up Supabase CLI linked to the cloud project so Claude can create and push database migrations automatically.

**Architecture:** Supabase CLI is initialized in the project root, linked to the cloud project via access token. New migrations go into `supabase/migrations/` and are pushed immediately with `supabase db push --linked`. A rule in AGENTS.md enforces this auto-push behavior.

**Tech Stack:** Supabase CLI, Supabase cloud (project ref: `planeljrbapkxdzbtjzm`), `.env.local` for credentials.

### Task 5: Add migration rule to AGENTS.md

**Files:**

- Modify: `AGENTS.md`

- [ ] **Step 1: Add migration section to AGENTS.md**

Append the following section to the end of `AGENTS.md`:

```markdown
# Database Migrations

When a database migration is needed:

1. Create migration file: `supabase migration new <descriptive_name>` (run from project root)
2. Write the SQL into the generated file at `supabase/migrations/YYYYMMDDHHMMSS_<name>.sql`
3. Push immediately: `SUPABASE_ACCESS_TOKEN=<token from .env.local> supabase db push --linked`
4. Report the result to the user

**Auto-push rule:** After creating any new migration file in `supabase/migrations/`, always run `supabase db push --linked` immediately without waiting to be asked. Report success or failure.

**Credentials:** Read `SUPABASE_ACCESS_TOKEN` from `.env.local`. Never commit this value.
```

- [ ] **Step 2: Commit**

```bash
git add AGENTS.md
git commit -m "docs: add supabase migration workflow rule to AGENTS.md"
```
